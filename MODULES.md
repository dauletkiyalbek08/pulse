## 0. Приоритет №1

Главное сейчас — **подключить базу данных (Supabase)** и довести до рабочего состояния:
каркас, авторизацию, «Мои проекты», создание проекта, дашборд education на демо-данных и
**базовые экраны модуля «Продажи и CRM»** + глобальный выбор диапазона дат.

Сложные AI-функции (агент рекламы, атрибуция, CAPI, оценка звонков, генераторы) описаны
здесь для контекста, но реализуются позже. Сейчас под них делаются аккуратные
разделы-заглушки с правильной структурой меню — без рабочей логики.

## 1. Глобально: выбор диапазона дат [СЕЙЧАС]

Один переиспользуемый компонент периода — используется на всех дашбордах и экранах аналитики.

Пресеты: **Сегодня, Вчера, Последние 7 дней, Последние 30 дней, Этот месяц, Прошлый месяц,
Прошлая неделя.** Плюс иконка календаря для **произвольного диапазона** (выбор начальной и
конечной даты).

По умолчанию — «Последние 7 дней». Выбранный период фильтрует все метрики на странице
(запросы к `metrics_daily`, `leads`, `sales` по дате). Компонент общий, не дублировать.

## 2. Модуль «Продажи и CRM»

### 2.1. Лиды [СЕЙЧАС]
Лид проходит этапы (статусы воронки). Карточка лида: имя, телефон, **источник**
(Instagram / Facebook / TikTok / другое), статус, ответственный (менеджер/хантер), дата,
сумма потенциальной сделки.

Подключение внешней CRM: если у клиента уже есть CRM (например **AmoCRM**) — предусмотреть
импорт лидов через интеграцию **[ПОЗЖЕ]**. Если CRM нет — лиды ведутся вручную внутри Pulse
**[СЕЙЧАС]**.

Лид, пришедший из чат-бота или формы с оставленным номером, **автоматически** попадает сюда
с пометкой источника (см. 4.2) **[ПОЗЖЕ]**.

### 2.2. CRM-воронка [СЕЙЧАС — базово]
Визуальная воронка по статусам. Для education: **Лид → Пробный урок → Продажа**; для
ecommerce: **Лид → Обработан → Продажа**. Отображение по колонкам (канбан). Перетаскивание
лида между этапами — **[ПОЗЖЕ]**, сейчас достаточно отображения и счётчиков по этапам.

### 2.3. Продажи [СЕЙЧАС]
Карточки сверху (все реагируют на выбранный диапазон дат):
**Доход** (выручка за период), **Расходы** (рекламный бюджет за период), **Чистая прибыль**,
**Количество продаж**, **Средний чек**, **Конверсия**.

Список продаж: дата, клиент, что купил (курс/товар), сумма, менеджер. Источник данных —
таблица `sales`; суммы и графики — из `metrics_daily` за выбранный период.

### 2.4. Клиенты [СЕЙЧАС]
Те, кто уже купил / учится. По каждому клиенту: имя, контакты, **какие курсы купил**,
**сумма каждой покупки**, **общая сумма (LTV)**, даты покупок. По сути — люди, у которых есть
продажи, агрегированные по человеку (один клиент мог купить несколько курсов).

Для этого добавить таблицу `customers` (см. раздел 6) и связать `sales.customer_id`.

### 2.5. Пробные уроки [СЕЙЧАС — education]
Список записавшихся на пробный урок: клиент, дата/время, статус (записан / пришёл / не
пришёл / купил после), ответственный. Это промежуточный этап воронки образования.

### 2.6. Анализ звонков [ПОЗЖЕ]
Если есть внешняя CRM/телефония (AmoCRM и т.п.) — подключаем и тянем записи звонков. Если
нет — предусмотреть подключение телефона / загрузку записей. **AI оценивает звонок**: было
ли приветствие, длительность, структура разговора, выявление потребности, закрытие — и
выдаёт **балл + краткий разбор**. Требует AI и телефонию → поздний этап. Сейчас — заглушка
со структурой экрана.

## 3. Модуль «Маркетинг» (почти всё — [ПОЗЖЕ], сейчас заглушки)

### 3.1. Реклама + AI-агент [ПОЗЖЕ]
Кнопка запуска рекламы. Цель — **AI-агент**, который сам создаёт, запускает и анализирует
кампании. Интеграция: **Meta Ads** (подключаем), **TikTok Ads** (пока под вопросом — оставить
место в архитектуре). Это поздний этап; сейчас раздел-заглушка.

### 3.2. Сводная аналитика / атрибуция [ПОЗЖЕ]
Когда человек покупает курс/товар — определить, **откуда он пришёл, какой креатив видел**,
весь путь взаимодействия с аккаунтом. Связка **креатив → лид → продажа**. Рейтинг креативов
по эффективности (CPL, конверсия, ROAS, доля качественных лидов) — чтобы видеть, какие
креативы дают качественных лидов и продажи. Потребует таблицу `creatives` и данные из Meta
(раздел 6). Поздний этап.

### 3.3. SMM Studio [ПОЗЖЕ]
Подключение AI-сервиса (Claude / DeepSeek и т.п.) для **генерации идей контента**: посты,
stories, карусели, под событие/инфоповод. SMM-специалист задаёт тему → получает готовые
идеи и тексты для аккаунта. Поздний этап; заглушка сейчас.

### 3.4. CAPI — Conversion API [ПОЗЖЕ]
Поток: менеджер продаёт курс и получает **чек об оплате** → отправляет чек в **Telegram-бот**
→ бот анализирует и фиксирует покупку (статус «подтверждённая покупка») → событие покупки
**автоматически уходит в Meta Ads через Conversion API**, чтобы Meta оптимизировала рекламу
на похожих покупателей. Поздний этап (нужны бот + Meta CAPI). Сейчас — описание потока и
заглушка.

### 3.5. Ресурсы / Воронки [ПОЗЖЕ]
Все наши **сайты и лендинги**, на которые льём трафик. Можно создавать временные сайты и
постепенно дорабатывать. Хранение ссылок/страниц и привязка к кампаниям. Поздний этап.

### 3.6. AI Studio [ПОЗЖЕ]
Подключение нейросетей для **создания креативов**: видео-креативы, баннеры. Генерирует
рекламные материалы. Поздний этап; заглушка сейчас.

## 4. Модуль «Автоматизация»

### 4.1. Интеграции [СЕЙЧАС — каркас]
Единый экран всех подключённых сервисов: Meta Ads, TikTok Ads, AmoCRM, Telegram,
AI-сервисы. Показывает статус подключения. **Ключи и токены хранятся только на сервере, в
зашифрованном виде** (см. CLAUDE.md, п. 3.6). Сейчас — каркас экрана и таблица `integrations`;
сами подключения добавляем по мере готовности.

### 4.2. ChatBot Builder [ПОЗЖЕ]
Подключаем к платформе. Видим, **кто нам писал** (Instagram Direct и т.п.). Если человек
оставил номер — он **автоматически попадает в «Лиды»** с указанием источника (например,
«пришёл из Instagram»). Поздний этап; заглушка сейчас.

## 6. Дополнения к схеме БД

### Сейчас [СЕЙЧАС]
```sql
-- Клиенты (купившие). Один клиент может иметь несколько продаж.
create table customers (
  id                uuid primary key default gen_random_uuid(),
  project_id        uuid not null references projects(id) on delete cascade,
  full_name         text not null,
  phone             text,
  first_purchase_at timestamptz,
  total_spent       numeric not null default 0,  -- LTV, пересчитывать при добавлении продажи
  created_at        timestamptz not null default now()
);
-- Добавить в таблицу sales колонку:
alter table sales add column customer_id uuid references customers(id);
```
Включить RLS на `customers` по тому же шаблону, что и для остальных таблиц проекта
(владелец — всё; участник проекта — только свои строки; см. CLAUDE.md, раздел 7).

### Позже [ПОЗЖЕ]
```sql
-- Подключённые интеграции (Meta, TikTok, AmoCRM, Telegram, AI-сервисы)
create table integrations (
  id          uuid primary key default gen_random_uuid(),
  project_id  uuid not null references projects(id) on delete cascade,
  provider    text not null,            -- 'meta' | 'tiktok' | 'amocrm' | 'telegram' | 'ai'
  status      text not null default 'disconnected',
  config      jsonb,                     -- НЕ хранить секреты открыто; токены — отдельно, зашифрованно
  created_at  timestamptz not null default now()
);

-- Креативы (для атрибуции креатив → лид → продажа)
create table creatives (
  id          uuid primary key default gen_random_uuid(),
  project_id  uuid not null references projects(id) on delete cascade,
  name        text not null,
  platform    text,                      -- 'meta' | 'tiktok'
  external_id text,                       -- id креатива в рекламном кабинете
  created_at  timestamptz not null default now()
);
-- Позже связать: leads.creative_id и sales.creative_id для сводной аналитики.
```
